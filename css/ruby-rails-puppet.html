Philosophy

  • Principle of Least Surprise
    • “I believe people want to express themselves when they program. They don’t want to fight with the language. Programming languages must feel natural to programmers. I tried to make people enjoy programming and concentrate on the fun and creative part of programming when they use Ruby.”
      • Yukihiro “Matz” Matsumoto, created ruby in 1993
    • http://www.linuxdevcenter.com/pub/a/linux/2001/11/29/ruby.html

Philosophies

  • Programming languages should be expressive for programmers
    • more than for computers
  • Code should be elegant
  • DRY (Don’t Repeat Yourself)
    • guides when to abstract
  • YAGNI (You Ain’t Gonna Need It)
    • guides when not to abstract
    • similar to DTSTTCPW (do the simplest thing that can possibly work)
  • TIAS (Try It And See)
    • Can a variable name have a question mark? It’s faster to test it out in irb than to google for the answer.

Interpreted

  • Not Compiled
    • variants like JRuby are
  • ruby interpreter evaluates a file
  • irb Interactive Ruby Shell
  • pry advanced ruby shell

Duck Typing

    • “If it walks and talks like a duck…”
    • Then it’s a duck!
    • As long as an object has the methods we expect, we can ignore its type.
    • Dynamic Typing: objects aren’t type checked until runtime
    • Objects are strongly typed
      • everything has a type and objects don’t change type
    • Types aren’t checked unless code explicitly checks

Output: puts & print

  • puts(str)
    • prints a string with a newline
    • ruby’s System.out.println
    • parentheses are optional
  • print(str)
    • like puts, but no newline

> print "Hello"
Hello => nil
  • The text is being printed out, but the actual calls to puts and print don’t return anything, so they evaluate to nil

Blocks

  • Similar in structure to if / unless / while
  • Denoted with do … end or { … }
  • Can take parameters, notated using pipes
    • | block_param1, block_param2 |
  • Functions can be passed blocks, giving ruby functional semantics
    • Array#each and Array#map are two of the most commonly-used functions that take blocks.

Array#each

# We already saw one way to iterate through an array
for day in [:mon, :tues, :wed]
  puts "#{day}"
end

# We can also do this by passing Array#each a block
[:mon, :tues, :wed].each do |day|
  puts "#{day}"
end
  • This block syntax and using each instead of for may feel awkward at first when coming from another language, but stick with it and you’ll find yourself among the ruby programmers who find it enjoyable to read.

Hash#each

# we can also use each to 
# loop through a hash's key/value pairs

map = {mon: 1, tues: 2, wed: 3}
map.each do |name, day|
  puts "#{name}is day #{day}of the week"
end

Functions

# arguments can have default values
# in case we don't specify them
def decrement(number, decrement_by = 1)
  number - decrement_by
end

# arguments can be any type, like arrays or hashes.
# how we check for valid inputs is up to us
def lookup(needles = [], haystack = {})
  # needles could be a non-array,
  # or haystack could be a non-hash,
  # in which case this will raise an exception
  needles.map do |needle|
    haystack.has_value?(needle)
  end
end

Accessors

  • These simple getters and setters are so common
  • And they take up a lot of space in our files
  • Fortunately, there’s a shortcut!
    • attr_reader - generates a default getter
    • attr_writer - generates a default setter
    • attr_accessor - generates both a getter and a setter
  • A coded getter/setter takes precedence over attr_*

Accessors

class MyFile
  attr_accessor :name, :contents
end

my_file = MyFile.new
my_file.name = 'data.txt'
my_file.contents = 'Hello, File!'
puts "#{my_file.name}: #{my_file.contents}"
my_file.methods.sort

Accessor + Custom Setter

  • Sometimes we want to perform custom logic in just a getter or setter.
class MyFile
  attr_reader :name

  def name=(filename)
    @name = filename.upcase
  end
end

my_file = MyFile.new
my_file.name = "data.txt"
puts "I just created #{my_file.name}"
    • Why did we call this class MyFile instead of File? It turns out that ruby already has a class named File, so this could have been confusing. We could have put this file into a module (e.g. RubyShell) and referenced it as RubyShell::File to get around this.

Multiple Inheritance?

    • Assume we have the following classes:
      • Script - represents some executable code and provides an execute method
      • MyFile - represents a file with a name and contents
    • Now we want to declare a ScriptFile that represents a file whose content is a script that can be executed.
    • class ScriptFile < Script, MyFile ?
      • × Not Valid
    • Two Options: encapsulation & mixins

Encapsulation

class ScriptFile < MyFile
  attr_reader :script

  def initialize(script)
    @script = script
  end

  # delegate the execute functionality to the Script
  def execute
    @script.execute
  end
end
    • Rails’ ActiveSupport module provides functionality so we can just say delegate :execute, to: :script, and Ruby has similar functionality via the Forwardable module

Mixin

module Script
  def execute
    # assumes whoever mixes this in will have a method `contents`
    eval(contents) # caution: eval can be dangerous!
  end
end

class ScriptFile < MyFile
  include Script
end

ScriptFile.ancestors
# => [ScriptFile, Script, MyFile, Object, Kernel, BasicObject]

script = ScriptFile.new
script.contents = "1+1"
script.execute #=> 2
script.is_a?(Script) #=> true

Access Modifiers

  • public (default) - can be called from anywhere
  • private - can only be called from within this class

  • protected - can only be called where self is of the same type

    • There are ways around these modifiers, so they’re mostly just hints about what’s safe to call
    • For a great explanation, see https://tenderlovemaking.com/2012/09/07/protected-methods-and-ruby-2-0.html

Access Modifiers

class MyTextFile
  def initialize
    @current_line = 0
    @lines = []
    10.times{ |i| @lines << "line #{i}" }
  end

  def readline
    return '' if end_of_file?
    line = @lines[@current_line]
    increment_line
    line
  end

private

  def increment_line
    @current_line += 1
  end

  def end_of_file?
    @current_line + 1 > @lines.length
  end
end

Test Driven Development

  1. Write basic test
    • failing (red)
    • shows how we want to use the code we’ll write
  2. Write just enough code to get test passing
    • basic passing (green)
  3. Write more failing tests (more complex functionality, edge cases)
    • failing (red)
  4. Implement functionality to get tests passing
    • passing (green)
  5. Repeat
  6. After time, clean up code / architecture while keeping tests passing
    • refactor
  • Red, Green, Refactor

  def update_puppet_facts
    game_servers = Config.all.map do |config|
      { "server" => "#{config.server_url}:#{config.server_port}",
        "game_type" => config.game_type }
    end
    yml = {"game_servers" => game_servers}.to_yaml

    File.open('/etc/puppet/code/hiera/common.yaml', 'w') do |f|
      f.puts yml
    end
  end
end

String Reference

  • Full reference: https://ruby-doc.org/core/String.html
  • Several of these methods have a ! version as well. The convention is that the ! version will modify the string you call it on, whereas the non-! version returns a new string.
  • + concatenate two strings (e.g. "thank " + "you" #=> "thank you")
  • == equality operator
  • =~ matching operator (e.g. "thank you" =~ /thank/)
  • capitalize converts the first character to upper case
  • chomp removes trailing whitespace
  • strip removes leading and trailing whitespace
  • upcase converts the string to all caps
  • downcase converts the string to all lowercase
  • empty? returns if the string has zero length
  • gsub replaces all occurrences of a pattern with a replacement
  • sub replaces the first occurrence of a pattern with a replacement
  • include? determines if the string contains the given substring
  • length the character length of the string
  • reverse the backwards version of the string
  • split splits the string based on the given pattern into an array of strings separated by that pattern
  • to_i converts the string into an integer of the given base (if possible)
  • to_sym converts the string into a symbol

Array Reference

  • Full reference: http://ruby-doc.org/core/Array.html
  • + joins two arrays
  • - is the difference between two arrays
  • & is the intersection of two arrays
  • <<, push adds to the end of an array
  • unshift adds to the beginning of the array
  • insert adds an element to a specific place in the array
  • <=> is the comparison operator
  • [index] retrieves the element at index
  • [start, length] and [start..end] retrieves a subarray
  • any? {|obj| block} returns true if the block returns true for any element
  • select {|obj| block} returns the elements for which the block returns true
  • map {|obj| block} returns a new array with the block being performed on each element
  • count, length returns the number of elements in the array
  • take(n) returns the first n elements of the array
  • drop(n) returns the tail of the array after skipping the first n elements
  • each {|obj| block} performs the block on each element of the array
  • each_with_index {|obj, idx| block} same as each but also gives access to the index
  • include? returns if the given element is in the array
  • pop removes the last element from the array and returns it
  • shift removes the first element from the array and returns it
  • reverse returns the array backwards
  • sample returns a random element (or n random elements) from the array
  • shuffle randomizes the array
  • sort sorts the array using <=> on each of the elements
  • join builds a string from the array elements

RSpec Matcher Reference

  • Full Reference: https://github.com/rspec/rspec-expectations
  • Equivalence
    • expect(actual).to eq(expected)
    • expect(actual).not_to eq(expected)
  • Identity
    • expect(actual).to be(expected)
    • expect(actual).to equal(expected)
  • Comparisons
    • expect(actual).to be > expected
    • expect(actual).to be <= expected
    • expect(actual).to be_within(delta).of(expected)
  • Regular Expressions
    • expect(actual).to match(/expression/)
    • e.g., expect("hello, world!").to match(/[a-z]+, [a-z]+!/)
  • Types/classes
    • expect("hello").to be_an_instance_of(String)
    • expect(3).to be_a(Numeric)
    • expect(3).to be_an(Integer)
  • Truthiness
    • expect(3).to be_truthy
    • expect(true).to be_true
    • expect(nil).to be_falsy
    • expect(false).to be_false
    • expect(nil).to be_nil
    • expect(3).to_not be_nil
  • Expecting errors
    • expect { raise NotImplementedError }.to raise_error(NotImplementedError)
  • Predicate matchers
    • expect(actual).to be_xxx # passes if actual.xxx?
    • expect(actual).to have_xxx(:arg) # passes if actual.has_xxx?(:arg)
  • Collection Membership
    • expect(actual).to include(expected)
    • expect(actual).to start_with(expected)
    • expect(actual).to end_with(expected)
